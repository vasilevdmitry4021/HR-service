# Руководство пользователя HR-сервиса

## Для кого этот сервис

HR-сервис помогает искать кандидатов через HeadHunter. Вы можете описывать требования обычными словами — система сама подберёт подходящие фильтры и покажет релевантные резюме.

---

## Начало работы

### Регистрация и вход

1. Откройте страницу **Регистрация**.
2. Введите email и пароль (не менее 8 символов).
3. Нажмите «Зарегистрироваться».
4. После регистрации вы попадёте на страницу поиска.
5. Для повторного входа используйте страницу **Вход** с теми же данными.

---

## Разделы системы

### Поиск

Главный раздел для поиска кандидатов.

**Как искать:**

1. Введите запрос в поле поиска. Можно писать свободно, например:
   - «Java-разработчик с опытом от 3 лет в Москве»
   - «Мне нужен сотрудник с опытом разработки на Python в ритейле»
   - «Найди мужчину 25–35 лет, backend-разработчик»
2. Система автоматически распознает:
   - регион
   - возраст
   - пол
   - навыки
   - отрасль
   - должность
   - опыт работы
3. При необходимости уточните параметры в панели фильтров справа.
4. Нажмите кнопку поиска.
5. Результаты появятся ниже с оценкой релевантности.

**Шаблоны поиска**

- Сохраняйте часто используемые запросы как шаблоны.
- Нажмите «Сохранить как шаблон», введите название и сохраните.
- Шаблоны можно применять одним нажатием «Применить».

**Кнопка «Только разобрать запрос»**

- Показывает, какие параметры система извлекла из текста, без выполнения поиска.

---

### Кандидаты

- Нажмите на карточку кандидата, чтобы открыть полное резюме.
- На странице резюме можно:
  - добавить кандидата в избранное;
  - скачать резюме в PDF (если функция включена).

---

### Избранное

Здесь хранятся отобранные кандидаты.

- Добавляйте кандидатов в избранное со страницы поиска или карточки резюме.
- Для каждого кандидата можно добавить заметки.
- Заметки сохраняются кнопкой «Сохранить заметки».
- Удалить кандидата из избранного можно кнопкой «Удалить».

---

### История поиска

- Все успешные поиски сохраняются.
- В таблице отображаются дата, запрос и количество найденных резюме.
- Кнопка «Повторить» открывает тот же запрос и фильтры на странице поиска.

---

### Настройки

Раздел для подключения к HeadHunter.

- Нажмите «Подключить HeadHunter», чтобы авторизоваться через OAuth.
- После подключения статус отображается в правом верхнем углу.
- Если включён тестовый режим (mock), подключение выполняется мгновенно для проверки.

---

## Подсказки

- Пишите запросы на русском языке.
- Чем подробнее описание, тем точнее подбор.
- Фильтры и текст запроса синхронизируются: изменения в одном месте отражаются в другом.
- Используйте шаблоны для типовых поисков.
- Релевантность показывает, насколько резюме подходит вашему запросу.

## Проверка работы в тестовом режиме (моки)

Если включены моковые LLM и HeadHunter (`FEATURE_USE_MOCK_LLM=true`, `FEATURE_USE_MOCK_HH=true`), система работает без реального API.

### Что уже работает

- **Распознавание запроса** — блок «Распознанные параметры» показывает, что mock LLM извлекает навыки, регион, опыт и т.п.
- **Поиск** — работает без подключения к HeadHunter. Токен не нужен.

### Почему может быть «Кандидаты не найдены»

В моке есть 4 тестовых резюме:

| Навыки | Регион | Опыт |
|--------|--------|------|
| Java, Spring Boot | Москва | 5 лет |
| Python, Django | Санкт-Петербург | 7 лет |
| JavaScript, React | Москва | 3 года |
| Go, Kubernetes | Екатеринбург | 4 года |

Если запрос и фильтры не совпадают с этими данными, результатов не будет. Например, «Java в Санкт-Петербурге» — в моке Java только в Москве.

### Как проверить поиск на моках

1. **Простой запрос** — введите, например:
   - «Java» или «Java Москва» — должен найти Иванова (Java, Москва)
   - «Python» или «Python Санкт-Петербург» — должен найти Петрову
   - «Go Екатеринбург» — должен найти Козлова
2. **Сбросьте конфликтующие фильтры** — если в панели справа выбран регион «Москва», а в тексте — «Санкт-Петербург», результат будет пустым. Выберите «Не задано» в регионе или приведите запрос и фильтры к одному региону.
3. **Нажмите «Найти»** — поиск выполняется даже при статусе «HH: not connected».

### Как сделать статус «HH: connected»

1. Перейдите в **Настройки**.
2. Нажмите **«Подключить HeadHunter»**.
3. В mock-режиме произойдёт мгновенный редирект, токен сохранится.
4. Статус сменится на «HH: connected».

## Работа с реальной LLM (Ollama)

При `FEATURE_USE_MOCK_LLM=false` запросы обрабатывает реальная LLM (Ollama или совместимый API).

### Диагностика

Если «Распознанные параметры» пустые:

1. **Проверьте модель** — Ollama возвращает `model 'llama3.2' not found`, если модель не установлена.
   - На сервере Ollama выполните: `ollama list` — посмотрите доступные модели.
   - Установите модель: `ollama pull llama3.2` или укажите в `.env` другую модель: `INTERNAL_LLM_MODEL=llama3` (или mistral, qwen2 и т.п.).
2. **Отладочный endpoint** — POST `/api/v1/search/parse/debug` с телом `{"query":"..."}` возвращает сырой ответ LLM:
   - `ok: true` — LLM ответила, смотрите `raw.response`;
   - `ok: false`, `error` — причина (сеть, 404, модель не найдена).

### Настройки (.env)

| Переменная | Описание |
|------------|----------|
| `FEATURE_USE_MOCK_LLM` | `false` — использовать реальную LLM |
| `INTERNAL_LLM_ENDPOINT` | URL, например `http://10.118.133.147:11434/api/generate` |
| `INTERNAL_LLM_MODEL` | Имя модели (должна быть установлена на сервере Ollama) |

## Частые вопросы

**Почему «Распознанные параметры» пустые при реальной LLM?**  
Чаще всего модель не установлена на Ollama. Выполните `ollama list` на сервере и укажите в `INTERNAL_LLM_MODEL` имя доступной модели или установите нужную: `ollama pull llama3.2`. Для диагностики используйте POST `/api/v1/search/parse/debug`.

**Почему нет результатов?**  
Попробуйте упростить запрос или ослабить фильтры. В тестовом режиме (моки) проверьте, что запрос и фильтры соответствуют тестовым данным (см. раздел «Проверка работы в тестовом режиме»).

**Как изменить пароль?**  
Сейчас смена пароля не реализована. Используйте текущие учётные данные.

**Где хранятся данные?**  
Данные хранятся на сервере и привязаны к вашему аккаунту.
