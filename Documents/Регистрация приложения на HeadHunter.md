# Регистрация приложения на HeadHunter

## Введение

Для работы с API HeadHunter необходимо зарегистрировать приложение на [dev.hh.ru](https://dev.hh.ru). Этот документ содержит готовые тексты для заполнения формы регистрации.

**Связанные документы:**
- [Roadmap.md](Roadmap.md)
- [Техническая спецификация.md](Техническая%20спецификация.md)

---

## Предварительные требования

| Требование | Описание |
|------------|----------|
| Аккаунт работодателя | Зарегистрироваться на [hh.ru](https://hh.ru) как работодатель |
| Авторизация на dev.hh.ru | Войти под аккаунтом работодателя |

> 💡 **Приложения ещё нет — это нормально!** HeadHunter понимает, что разработка начинается после получения доступа к API. Redirect URI можно указать ориентировочный, после утверждения заявки его можно отредактировать.

## Заполнение формы

### 1. Информация о приложении

| Поле | Что указать |
|------|-------------|
| **Название приложения** | `HR-сервис поиска кандидатов` |
| **Redirect URI** | `http://localhost:8000/api/v1/hh/callback` |

> На этапе разработки укажите localhost. После деплоя обновите на production URL, например: `https://hr-service.example.com/api/v1/hh/callback`

Приложение предназначено исключительно для внутреннего использования

Приложение предназначено для упрощения и ускорения процесса подбора

формулировать запросы на естественном языке вместо ручного заполнения

### 5. Функциональные возможности

#### Опишите все функциональные возможности приложения и укажите используемые методы API

```
ФУНКЦИОНАЛЬНЫЕ ВОЗМОЖНОСТИ:

1. Авторизация через OAuth 2.0
   - Получение access_token и refresh_token
   - Автоматическое обновление токенов

2. Поиск резюме
   - Поиск по ключевым словам и фильтрам
   - Фильтрация по региону, опыту, навыкам, возрасту, полу

3. Просмотр резюме
   - Получение детальной информации о кандидате
   - Просмотр контактных данных (при наличии оплаченной услуги)

4. Работа со справочниками
   - Получение списка регионов, должностей, отраслей, навыков

ИСПОЛЬЗУЕМЫЕ МЕТОДЫ API:

- GET /resumes — поиск резюме
- GET /resumes/{resume_id} — получение резюме
- GET /areas — справочник регионов
- GET /professional_roles — справочник должностей
- GET /industries — справочник отраслей
- GET /skills — справочник навыков
- GET /employer/managers — проверка токена
- POST /oauth/token — получение/обновление токенов

СОБЛЮДЕНИЕ УСЛОВИЙ ИСПОЛЬЗОВАНИЯ:

- Приложение НЕ хранит данные резюме дольше сессии пользователя
- Приложение НЕ передаёт данные третьим лицам
- Приложение НЕ выполняет действия без ведома пользователя
- Приложение НЕ собирает логины/пароли пользователей HH
- Все запросы выполняются от имени авторизованного пользователя
```

## После отправки заявки

| Этап | Описание | Срок |
|------|----------|------|
| Рассмотрение | HeadHunter проверяет заявку | до 15 рабочих дней |
| Одобрение | Получение `client_id` и `client_secret` | — |
| Настройка | Обновление Redirect URI на production | — |

---

## После одобрения

1. **Получите credentials** — `client_id` и `client_secret` появятся в личном кабинете dev.hh.ru
2. **Обновите Redirect URI** — можно добавить несколько URL (localhost для разработки + production)
3. **Сохраните безопасно** — добавьте в переменные окружения, никогда не коммитьте в Git!

```bash
# .env (пример)
HH_CLIENT_ID=your_client_id
HH_CLIENT_SECRET=your_client_secret
HH_REDIRECT_URI=https://hr-service.example.com/api/v1/hh/callback
```

## Возможные причины отказа

| Причина | Как избежать |
|---------|--------------|
| Неполное описание | Заполните все поля подробно |
| Публичный сервис без обоснования | Выберите "Только сотрудники работодателя" |
| Нарушение условий использования | Укажите соблюдение всех правил |
| Отсутствие контактных данных | Укажите реальный телефон и email |

> ⚠️ HeadHunter вправе отказать без объяснения причин. Обжалование возможно в течение 15 календарных дней.
