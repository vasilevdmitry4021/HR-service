# Допущение: хардкод логина e-staff при создании кандидата

## Суть

В коде сервиса временно задан логин пользователя e-staff (`user_login`), который подставляется при каждом вызове API добавления кандидата к вакансии. Это сделано для тестового доступа, где без явной привязки кандидаты не отображаются в интерфейсе из‑за ограничений учётной записи.

Реализация: константа `_ESTAFF_HARDCODED_CANDIDATE_USER_LOGIN` в модуле `backend/app/services/estaff_client.py`.

## Требование к продуктиву

Перед выкладкой в продуктив **необходимо убрать хардкод**: удалить подстановку `user_login` из тела запроса к e-staff (или заменить на настраиваемое поле в UI/конфигурации, если так решит команда).

Иначе все кандидаты, добавленные через интеграцию, будут числиться за одним и тем же пользователем e-staff, что для боевых HR-корректно не является.
