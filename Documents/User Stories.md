# User Stories: HR-сервис поиска кандидатов

## Введение

**Проект:** HR-сервис интеграции с HeadHunter

**Назначение:** Сервис для поиска кандидатов через API HeadHunter с использованием естественного языка и интеллектуальной обработки запросов.

**Связанные документы:** [Требования.md](Требования.md)

---

## Эпик 1: Управление доступом и аутентификация

### US-1.1: Настройка API-ключа

> Как HR-специалист, я хочу ввести и сохранить API-ключ от HeadHunter, чтобы система могла выполнять запросы к базе резюме от моего имени.

**Критерии приёмки:**
- Пользователь может ввести API-ключ в защищённое поле
- Ключ сохраняется в зашифрованном виде
- Система отображает статус валидности ключа (действителен/недействителен)
- При невалидном ключе отображается понятное сообщение об ошибке

---

### US-1.2: Проверка подключения

> Как HR-специалист, я хочу видеть статус подключения к HeadHunter, чтобы понимать, работает ли интеграция.

**Критерии приёмки:**
- Индикатор статуса подключения виден на главном экране
- При потере связи отображается уведомление
- Есть кнопка "Проверить подключение"

---

## Эпик 2: Поиск кандидатов через текстовый запрос

### US-2.1: Ввод запроса на естественном языке

> Как HR-специалист, я хочу описать требования к кандидату в свободной форме, чтобы быстро начать поиск без изучения сложных фильтров.

**Критерии приёмки:**
- Поле ввода поддерживает многострочный текст
- Система принимает запросы на русском языке
- Есть примеры запросов для помощи пользователю
- Запрос можно отправить по кнопке или клавишей Enter

**Пример запроса:**
```
Мне нужен сотрудник с опытом разработки на Java в ритейле
```

---

### US-2.2: Автоматическое распознавание критериев из текста

> Как HR-специалист, я хочу, чтобы система автоматически понимала мои требования из текстового описания, чтобы не заполнять фильтры вручную.

**Критерии приёмки:**
- Система распознаёт из текста: регион, возраст, пол, навыки, отрасль, должность, опыт работы
- Распознанные критерии отображаются визуально (например, подсвечиваются в тексте)
- Пользователь видит, какие параметры были извлечены

**Пример:**
| Текст запроса | Распознанные критерии |
|---------------|----------------------|
| "Найди мне мужчину с опытом разработки на Java в ритейле" | Пол: мужской, Навыки: Java, Отрасль: ритейл, Должность: разработчик |

---

## Эпик 3: Работа с фильтрами

### US-3.1: Ручная настройка фильтров

> Как HR-специалист, я хочу выбирать критерии поиска через стандартные элементы управления (чекбоксы, выпадающие списки), чтобы точно указать требования.

**Критерии приёмки:**

| Фильтр | Тип элемента управления |
|--------|------------------------|
| Регион | Выпадающий список с поиском |
| Пол | Радиокнопки (мужской / женский / любой) |
| Возраст | Два поля ввода (от / до) |
| Опыт работы | Выпадающий список или слайдер |
| Должность | Поле с автодополнением |
| Навыки | Мультивыбор с тегами |
| Отрасль | Выпадающий список с поиском |

---

### US-3.2: Синхронизация текста и фильтров (текст → фильтры)

> Как HR-специалист, я хочу, чтобы при вводе текстового запроса соответствующие фильтры заполнялись автоматически, чтобы видеть структурированное представление моего запроса.

**Критерии приёмки:**
- При вводе "мужчина" автоматически выбирается фильтр "Пол: мужской"
- При вводе "Java" добавляется навык Java
- При вводе "ритейл" выбирается соответствующая отрасль
- Заполнение происходит в реальном времени ("на лету")
- Пользователь может отменить автоматическое заполнение

---

### US-3.3: Обработка множественных соответствий

> Как HR-специалист, я хочу видеть все возможные варианты интерпретации моего запроса, чтобы выбрать наиболее подходящий.

**Критерии приёмки:**
- При неоднозначном запросе система предлагает варианты
- Пользователь может выбрать один или несколько вариантов
- Выбор сохраняется для последующих запросов (обучение предпочтениям)

**Пример:**
```
Запрос: "разработчик Java"

Предложенные варианты:
☑ Backend-разработчик
☑ Java-разработчик
☐ Full-stack разработчик
```

---

### US-3.4: Синхронизация фильтров и текста (фильтры → текст)

> Как HR-специалист, я хочу, чтобы при изменении фильтров текстовое описание запроса обновлялось, чтобы сохранять целостную картину поиска.

**Критерии приёмки:**
- Изменение фильтра отражается в текстовом поле (опционально)
- Пользователь может включить/выключить эту синхронизацию

---

## Эпик 4: Результаты поиска

### US-4.1: Просмотр списка кандидатов

> Как HR-специалист, я хочу видеть список найденных кандидатов с ключевой информацией, чтобы быстро оценить релевантность.

**Критерии приёмки:**
- Отображается: ФИО, текущая должность, опыт работы, ключевые навыки
- Есть индикатор соответствия запросу (%)
- Поддерживается пагинация или бесконечная прокрутка
- Отображается общее количество найденных кандидатов
- Круговая диаграмма или прогресс-бар показывает общий балл релевантности
- При клике на индикатор релевантности раскрывается детализация по категориям (навыки, опыт, отрасль, должность, регион)
- Цветовая индикация релевантности:
  - Зелёный: ≥80% (strong_match)
  - Жёлтый: 50-79% (good_match / partial_match)
  - Красный: <50% (weak_match)

---

### US-4.2: Просмотр детальной информации о кандидате

> Как HR-специалист, я хочу открыть полное резюме кандидата, чтобы изучить его опыт и квалификацию.

**Критерии приёмки:**
- Открывается карточка с полной информацией
- Есть ссылка на оригинальное резюме на HeadHunter
- Можно скачать резюме в PDF

---

### US-4.3: Сохранение кандидатов в избранное

> Как HR-специалист, я хочу сохранять интересных кандидатов в избранное, чтобы вернуться к ним позже.

**Критерии приёмки:**
- Кнопка "Добавить в избранное" на карточке кандидата
- Отдельный раздел "Избранное" со списком сохранённых
- Возможность удалить из избранного

---

### US-4.4: Добавление заметок к кандидату

> Как HR-специалист, я хочу добавлять заметки к сохранённым кандидатам, чтобы фиксировать важную информацию о них (результаты собеседования, впечатления, статус).

**Критерии приёмки:**
- Поле для ввода заметки в карточке избранного кандидата
- Автосохранение заметки при вводе (или кнопка "Сохранить")
- Заметка видна в списке избранного
- Поддержка многострочного текста
- Дата последнего изменения заметки

**Пример использования:**
```
Кандидат: Иванов Иван (Java Developer)
Заметка: "Провели техническое интервью 15.01. 
Хорошие знания Spring Boot, слабый SQL. 
Готов к переезду. Ожидания по ЗП: 250к.
Статус: ждём решения руководителя."
```

---

### US-4.5: Оценка релевантности кандидата

> Как HR-специалист, я хочу видеть оценку соответствия каждого кандидата моему запросу, чтобы быстрее находить наиболее подходящих кандидатов и не тратить время на нерелевантные резюме.

**Критерии приёмки:**
- Каждый кандидат в результатах поиска имеет общий процент соответствия (0-100%)
- Есть детализация по категориям:
  - Навыки (skills_match)
  - Опыт работы (experience_match)
  - Отрасль (industry_match)
  - Должность (position_match)
  - Регион (location_match)
- Есть текстовое краткое обоснование оценки
- Рекомендация отображается визуально: "Отличный кандидат", "Хороший кандидат", "Частичное соответствие", "Слабое соответствие"
- Можно задать минимальный порог релевантности для фильтрации (по умолчанию 30%)
- Результаты отсортированы по убыванию релевантности

**Пример отображения:**
```
┌─────────────────────────────────────────────────────┐
│  Иванов Иван | Java Developer           [87%] 🟢   │
├─────────────────────────────────────────────────────┤
│  Навыки: ████████████░░ 95%                         │
│  Опыт:   ██████████░░░░ 85%                         │
│  Отрасль: ████████░░░░░░ 70%                        │
│  Должность: ███████████░░ 90%                       │
│  Регион: ██████████████ 100%                        │
├─────────────────────────────────────────────────────┤
│  "Отличное соответствие по навыкам Java и Spring    │
│   Boot, опыт 5 лет превышает требуемые 3 года."     │
└─────────────────────────────────────────────────────┘
```

## Эпик 5: История и шаблоны

### US-5.1: История запросов

> Как HR-специалист, я хочу видеть историю своих поисковых запросов, чтобы быстро повторить предыдущий поиск.

**Критерии приёмки:**
- Сохраняются последние 20-50 запросов
- Можно повторить запрос одним кликом
- Отображается дата и количество найденных результатов

---

### US-5.2: Сохранение шаблонов поиска

> Как HR-специалист, я хочу сохранять часто используемые комбинации фильтров как шаблоны, чтобы экономить время на типовых поисках.

**Критерии приёмки:**
- Можно сохранить текущий набор фильтров с названием
- Шаблоны доступны в выпадающем списке
- Можно редактировать и удалять шаблоны

---

## Эпик 6: Обработка ошибок и edge cases

### US-6.1: Обработка пустых результатов

> Как HR-специалист, я хочу получать подсказки при отсутствии результатов, чтобы скорректировать свой запрос.

**Критерии приёмки:**
- При 0 результатов система предлагает ослабить критерии
- Показываются конкретные рекомендации (например, "Расширьте регион поиска")

---

### US-6.2: Обработка ошибок API

> Как HR-специалист, я хочу понимать причину ошибки при сбое поиска, чтобы знать, как её исправить.

**Критерии приёмки:**
- Понятные сообщения об ошибках на русском языке
- Различаются ошибки сети, лимитов API, авторизации
- Есть рекомендации по устранению
